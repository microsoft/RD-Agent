# AutoRL-Bench 任务说明

你是一个强化学习训练 Agent，目标是通过 Post-Training 提升模型表现。

## 环境变量
- TASK: 任务名称
- MODEL_PATH: 基础模型路径
- DATA_PATH: 训练数据路径
- OUTPUT_DIR: 输出目录
- GRADING_SERVER_URL: 评测服务地址

## 目录结构
```
$WORKSPACE/
├── data/           # 训练数据（只读）
├── models/         # 基础模型（只读）
├── output/         # 输出目录
├── instructions.txt
└── description.md
```

## 任务流程
1. 阅读 description.md 了解任务目标和数据格式
2. 设计训练方案（reward 函数、超参数等）
3. 使用 $DATA_PATH 训练模型
4. 保存到 $OUTPUT_DIR
5. 提交评测: POST $GRADING_SERVER_URL/submit

## 训练方式

### 方式一：使用 TRL 框架（推荐）
适用于大多数 LLM Post-Training 任务：
```python
from trl import GRPOTrainer, GRPOConfig

def reward_func(completions, answer, **kwargs):
    # 定义 reward 逻辑
    return [1.0 if correct else -1.0 for ...]

trainer = GRPOTrainer(
    model=MODEL_PATH,
    reward_funcs=reward_func,
    args=GRPOConfig(...),
    train_dataset=dataset,
)
trainer.train()
trainer.save_model(OUTPUT_DIR)
```

### 方式二：自定义 Rollout
适用于交互式环境（如 ALFWorld）：
```python
# 自己实现 sampling → reward → update 循环
for epoch in range(num_epochs):
    # 1. Rollout: 生成样本
    outputs = model.generate(prompts)
    # 2. Reward: 计算奖励
    rewards = compute_rewards(outputs, ...)
    # 3. Update: 更新模型
    loss = policy_loss(outputs, rewards)
    optimizer.step()
```

## API
```bash
# 提交评测
curl -X POST "$GRADING_SERVER_URL/submit" -H "Content-Type: application/json" -d '{"model_path": "'$OUTPUT_DIR'"}'

# 查看最高分
curl "$GRADING_SERVER_URL/best"
```

## 注意
- 可多次提交，系统记录最高分
- 合理利用时间
- **重要**: trl 保存模型后，`tokenizer_config.json` 中的 `extra_special_tokens` 会被保存为 list 格式，但 vLLM/transformers 加载时需要 dict 格式。保存模型后需删除该字段，否则评测会失败。
